一、什么是嵌入式开发

嵌入式开发是围绕特定硬件设备进行的软件开发。和普通 PC 或服务器开发相比，嵌入式更关注硬件资源、实时性、功耗、启动速度、稳定性和长期运行。

  • 智能家居设备。
  • 车载设备。
  • 摄像头。
  • 网关。
  • 可穿戴设备。

一、网络编程先理解什么

Linux 网络编程的核心是 socket。socket 是应用程序访问网络的接口，HTTP、WebSocket、RPC 等上层协议最终都建立在 socket 之上。

入门需要先理解：

  • IP：主机地址。
  • 端口：进程提供服务的入口。
  • TCP：可靠字节流。
  • UDP：无连接数据报。
  • 服务端：监听端口，等待连接。
  • 客户端：主动连接服务端。

一、系统编程学什么

Linux 系统编程关注应用程序和操作系统之间的接口。普通业务代码更多关心业务逻辑，系统编程则关心程序如何使用文件、进程、线程、信号、内存和网络。

入门阶段重点掌握：

  • 文件描述符。
  • open、read、write、close。
  • 进程创建和等待。
  • 线程创建和同步。
  • 信号处理。
  • 错误码和 errno。

一、为什么先学命令行

Linux 应用开发离不开命令行。无论是编译、部署、查看日志、定位进程、分析网络，最终都会回到终端。命令行不是为了炫技，而是为了让你能直接控制系统。

入门 Linux 开发，建议先熟悉这些能力：

  • 能在目录中快速移动和查找文件。
  • 能查看文件内容和日志。
  • 能理解权限、用户、进程和环境变量。
  • 能把常用命令组合成脚本。
  • 能读懂构建脚本、启动脚本和部署脚本。

一、Linux 应用开发是什么

Linux 应用开发通常指在 Linux 系统上编写、编译、调试、运行和发布用户态程序。它和 Android、嵌入式、服务端开发都有交集，但核心基础基本一致：

  • 能把程序打包、部署，并在目标环境中稳定运行。

一、Codex CLI 是什么

Codex CLI 是 OpenAI Codex 的命令行工具，适合在本地终端里处理真实代码仓库中的开发任务。它可以读取当前项目、执行命令、修改文件、运行测试、解释代码、做代码审查，并且会根据权限配置决定什么时候直接执行、什么时候向你确认。

简单理解：

  • 在 IDE 中写代码时，可以把 Codex CLI 当成一个“终端里的结对编程助手”。
  • 在已有项目中排查问题时，可以让它先阅读代码，再给出修改方案并直接落地。
  • 在重复性任务中，可以用非交互命令让它批量完成检查、重构、生成文档等工作。
