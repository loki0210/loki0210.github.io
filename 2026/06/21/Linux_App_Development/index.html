一、Linux 应用开发是什么

Linux 应用开发通常指在 Linux 系统上编写、编译、调试、运行和发布用户态程序。它和 Android、嵌入式、服务端开发都有交集，但核心基础基本一致：

  • 熟悉 Linux 文件系统、权限、进程、线程、信号、网络等机制。
  • 能使用 GCC、Make、CMake 等工具构建程序。
  • 能使用 GDB、strace、lsof、top、journalctl 等工具定位问题。
  • 能理解动态库、静态库、环境变量、系统调用和进程通信。
  • 能把程序打包、部署，并在目标环境中稳定运行。

Linux 下既可以开发命令行工具，也可以开发后台服务、网络程序、桌面应用、驱动配套工具、嵌入式应用等。入门阶段建议从命令行程序和后台服务开始，因为它们能覆盖最多基础能力。

二、开发环境准备

2.1 选择发行版

常见选择：

  • Ubuntu / Debian：资料多，适合入门和服务器开发。
  • Fedora：软件较新，适合跟进新工具链。
  • CentOS Stream / Rocky Linux / AlmaLinux：偏企业服务器环境。
  • Arch Linux：滚动更新，适合熟悉 Linux 后使用。

如果是新手，建议使用 Ubuntu LTS。

9.2 信号

Linux 通过信号通知进程发生了某些事件。

常见信号：

  • SIGINT：终端 Ctrl+C。
  • SIGTERM：请求进程正常退出。
  • SIGKILL：强制杀死进程，不能被捕获。
  • SIGHUP：终端断开或请求重新加载配置。

十六、学习路线

建议按这个顺序学习：

  1. Linux 基础命令：cd、ls、cp、mv、rm、find、grep、sed、awk。
  2. Shell 基础：变量、管道、重定向、脚本、退出码。
  3. C 语言基础：指针、结构体、内存管理、文件 IO。
  4. 编译工具：GCC、Makefile、CMake。
  5. 调试工具：GDB、strace、lsof、valgrind。
  6. 系统编程：进程、线程、信号、IPC、socket。
  7. 服务化：日志、配置、systemd、权限隔离。
  8. 发布部署：压缩包、动态库依赖、deb/rpm、容器。
  9. 性能优化：top、perf、火焰图、内存分析。
  10. 工程化：单元测试、CI、代码规范、版本管理。

十七、入门项目练习

可以从这些小项目开始：

  • 写一个命令行计算器。
  • 写一个日志切分工具。
  • 写一个文件搜索工具。
  • 写一个 TCP echo server。
  • 写一个 HTTP 静态文件服务。
  • 写一个定时执行任务的小服务。
  • 写一个读取配置文件并输出日志的后台进程。

每个项目都尽量做到：

  • 支持命令行参数。
  • 有清晰的错误提示。
  • 有 README。
  • 可以使用 Makefile 或 CMake 构建。
  • 可以用 GDB 调试。
  • 可以打包发布。
