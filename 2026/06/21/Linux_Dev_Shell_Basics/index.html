一、为什么先学命令行

Linux 应用开发离不开命令行。无论是编译、部署、查看日志、定位进程、分析网络，最终都会回到终端。命令行不是为了炫技，而是为了让你能直接控制系统。

入门 Linux 开发，建议先熟悉这些能力：

  • 能在目录中快速移动和查找文件。
  • 能查看文件内容和日志。
  • 能理解权限、用户、进程和环境变量。
  • 能把常用命令组合成脚本。
  • 能读懂构建脚本、启动脚本和部署脚本。

rm -rf 会递归删除目录，不会进入回收站。开发中通常只用它删除构建目录、临时目录，不要在不确定路径时执行。

九、入门检查清单

掌握以下内容后，再继续学习 C/C++ 构建、系统调用和网络编程会顺很多：

  • 能用 cd、ls、find、rg 找文件。
  • 能用 cat、less、tail -f 看日志。
  • 能理解 >、>>、2>&1。
  • 能设置和读取环境变量。
  • 能写简单 Shell 脚本。
  • 能用 ps、kill 管理进程。
  • 能看懂项目里的 build.sh、run.sh、deploy.sh。

Linux 开发的第一步不是写复杂程序，而是能在系统里高效移动、观察和控制。
