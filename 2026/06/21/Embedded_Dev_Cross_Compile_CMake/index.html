一、什么是交叉编译

交叉编译是指在一种平台上编译另一种平台运行的程序。比如在 x86_64 Ubuntu 电脑上编译 ARM 开发板运行的程序。

常见组合：

为什么需要交叉编译：

  • 开发板性能弱，编译慢。
  • 开发板存储小，不适合安装完整工具链。
  • CI/CD 通常运行在服务器上。
  • 需要统一构建环境。

八、sysroot

sysroot 是目标系统根目录的一个镜像，里面包含目标板的头文件和库。

常见来源：

  • SDK 提供。
  • 从开发板复制。
  • Buildroot / Yocto 生成。

使用 sysroot：

编译器会从 sysroot 里找目标平台的头文件和库，避免错误链接到开发机库。

九、部署方式

常见方式：

  • scp 复制单个程序。
  • 打包 tar.gz。
  • 制作 deb/rpm 包。
  • 放进 rootfs 镜像。
  • OTA 升级。

入门阶段建议先用 scp，流程简单，便于快速验证。

稳定后再考虑系统化部署。

十、常见问题

10.1 Exec format error

10.2 GLIBC 版本不匹配

开发机工具链生成的程序依赖较新 glibc，目标板系统太旧。

解决：

  • 使用板厂 SDK。
  • 使用和目标板一致的 sysroot。
  • 使用 Buildroot/Yocto 统一生成工具链和系统。

嵌入式交叉编译的核心不是会敲一条编译命令，而是保证工具链、目标架构、sysroot、动态库和部署环境一致。
