一、嵌入式调试的层次

嵌入式调试比普通应用复杂，因为问题可能出在硬件、启动链路、内核、驱动、文件系统或应用。

常见调试手段：

  • 串口日志。
  • LED 状态灯。
  • JTAG/SWD。
  • OpenOCD。
  • GDB。
  • gdbserver。
  • 逻辑分析仪。
  • 示波器。

入门阶段先掌握串口、gdbserver，再逐步学习 JTAG/OpenOCD。

如果完全没有串口输出，优先检查硬件连接和波特率。

三、LED 状态灯

在早期启动或系统异常时，日志可能不可用。LED 是低成本状态指示。

常见设计：

  • 上电常亮：电源正常。
  • U-Boot 闪烁：bootloader 运行。
  • Kernel 慢闪：内核启动。
  • App 心跳：业务正常。
  • 快闪：错误状态。

不要小看 LED。量产设备没有屏幕时，状态灯是现场排障的重要线索。

五、JTAG/SWD

JTAG/SWD 可以在 CPU 层面调试，常用于：

  • 裸机程序。
  • Bootloader。
  • RTOS。
  • 早期启动问题。
  • 内核调试。

常见调试器：

  • J-Link。
  • ST-Link。
  • CMSIS-DAP。
  • FT2232。

JTAG/SWD 能做：

  • 下载程序。
  • 设置断点。
  • 单步执行。
  • 查看寄存器。
  • 查看内存。
  • 停止 CPU。

七、逻辑分析仪

逻辑分析仪适合看数字信号。

常见用途：

  • UART 波形。
  • I2C 地址和数据。
  • SPI 时钟和数据。
  • GPIO 翻转时序。

当软件日志显示“读不到设备”时，逻辑分析仪可以确认总线上到底有没有信号。

八、示波器

示波器适合看模拟电压和信号质量。

常见用途：

  • 电源纹波。
  • 上电时序。
  • 复位信号。
  • 时钟波形。
  • 电平是否达标。

如果设备偶发重启、通信不稳定，不要只看代码，电源和信号质量也可能是根因。

九、排错顺序

建议顺序：

  1. 电源是否正常。
  2. 串口是否有输出。
  3. Bootloader 是否运行。
  4. Kernel 是否启动。
  5. RootFS 是否挂载。
  6. 应用是否启动。
  7. 外设总线是否有信号。
  8. 驱动是否 probe。
  9. 应用协议是否正确。

不要跳过硬件基础检查。很多“软件问题”最后都是线接错、电压不对、复位没拉起来。

十、总结

嵌入式调试要分层。应用问题用日志和 gdbserver，启动问题看串口和 U-Boot，裸机和早期初始化问题用 JTAG/OpenOCD，外设问题用逻辑分析仪和示波器。

能把这些工具组合起来，嵌入式问题定位效率会明显提升。
