一、什么是 RTOS

RTOS 是 Real-Time Operating System，实时操作系统。它常用于单片机和资源受限设备中，用来管理多个任务、定时器、同步和通信。

常见 RTOS：

  • FreeRTOS。
  • RT-Thread。
  • Zephyr。
  • ThreadX。

RTOS 不等于 Linux。RTOS 更轻量，更关注实时性；Linux 功能更完整，但实时性需要额外配置才能增强。

二、为什么需要 RTOS

项目变复杂后会遇到：

  • 某个任务阻塞影响其它任务。
  • 定时逻辑混乱。
  • 外设事件难管理。
  • 通信和采集耦合严重。

RTOS 可以把不同功能拆成任务。

四、调度

RTOS 根据优先级和任务状态调度。

基本原则：

  • 高优先级任务优先运行。
  • 阻塞任务不会占 CPU。
  • 延时、等待队列、等待信号量都会让任务进入阻塞。

不要在高优先级任务里长时间死循环，否则低优先级任务可能得不到执行。

六、信号量

信号量常用于事件通知或资源计数。

二值信号量：

  • 中断通知任务。
  • 任务等待某个事件。

计数信号量：

  • 管理多个相同资源。
  • 统计事件数量。

例如 UART 接收中断释放信号量，串口处理任务被唤醒后读取数据。

九、常见问题

9.1 栈溢出

任务栈太小会导致异常。

处理：

  • 增大任务栈。
  • 减少局部大数组。
  • 检查 RTOS 栈水位。

9.2 优先级反转

低优先级任务持有锁，高优先级任务等待锁，中优先级任务一直运行，导致高优先级任务迟迟无法执行。

处理：

  • 使用支持优先级继承的互斥锁。
  • 缩短持锁时间。

9.3 死锁

多个任务互相等待资源。

处理：

  • 统一加锁顺序。
  • 减少嵌套锁。
  • 加超时。
