一、为什么要服务化

一个程序能在终端运行，不代表适合部署。真正部署到 Linux 设备或服务器时，需要解决：

  • 开机自动启动。
  • 异常退出自动重启。
  • 统一查看日志。
  • 使用非 root 用户运行。
  • 控制启动顺序。
  • 优雅停止。

Linux 现代发行版通常使用 systemd 管理服务。

四、优雅停止

停止服务：

systemd 默认会发送 SIGTERM。程序应该捕获 SIGTERM，完成资源清理后退出。

应用中需要处理：

  • 停止接收新请求。
  • 保存必要状态。
  • 关闭文件。
  • 关闭 socket。
  • 等待工作线程退出。

不要依赖 kill -9，SIGKILL 无法清理资源。

十、总结

服务化不是把程序丢到后台运行。一个合格的 Linux 服务需要可启动、可停止、可重启、可观察、可排错。

systemd 是部署 Linux 应用的基础工具。掌握它以后，应用从“能跑”到“能长期稳定运行”会跨过一大步。
