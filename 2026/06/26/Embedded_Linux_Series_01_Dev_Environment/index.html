嵌入式 Linux 入门的第一步，不是改内核，也不是写驱动，而是把开发环境搭起来，把板子稳定连上，把启动日志完整拿到。只要这一步不扎实，后面遇到启动失败、网络不通、应用跑不起来时，就会缺少最基本的排查入口。

一、准备哪些东西

最小开发环境包括：

  • 一台 Linux 开发机，Ubuntu、Debian、Arch 都可以。
  • 一块开发板或产品板。
  • 串口线，常见是 USB 转 TTL。
  • 网线或 USB 网络。
  • 电源和稳定的供电线。
  • 官方 SDK、镜像、烧录工具和文档。

开发机建议优先使用 Linux。Windows 也能开发，但涉及串口、NFS、TFTP、交叉编译和脚本时，Linux 环境更直接。

二、先接串口

串口是嵌入式开发最重要的调试入口。网络不通时还能看串口，系统进不了 shell 时还能看串口，内核 panic 时也通常只能靠串口日志。

三、保存启动日志

第一次启动板子时，不要只看屏幕滚动。建议把完整日志保存下来。

日志里重点标记：

  • U-Boot 版本。
  • DRAM 初始化容量。
  • Kernel 加载地址。
  • Device Tree 加载地址。
  • kernel command line。
  • rootfs 挂载方式。
  • init 进程启动结果。

后续系统起不来时，可以和这份基准日志对比。

五、建立工作目录

开发机上建议把资料分开：

不要把官方 SDK、自己写的应用、临时日志、烧录镜像混在一个目录。嵌入式项目经常要回溯某次固件对应的日志和配置，目录清晰能省很多时间。

六、第一天要达成什么

第一天不需要写代码，目标是：

  1. 串口能稳定连接。
  2. 启动日志能完整保存。
  3. 能进入 shell。
  4. 能确认 CPU、内存、rootfs 挂载情况。
  5. 开发机能和板子互相 ping 通。
  6. 能把一个文件传到板子。
