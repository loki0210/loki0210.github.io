一、为什么要重视内存

Linux 应用运行时间越长，内存问题越容易暴露。短时间测试正常，不代表长期运行稳定。内存泄漏、越界访问、重复释放、use-after-free 都可能导致崩溃、性能下降或系统被 OOM killer 杀死。

常见内存问题：

  • 内存泄漏。
  • 空指针访问。
  • 数组越界。
  • 重复释放。
  • 释放后继续使用。
  • 栈空间不足。
  • 内存碎片。

一、什么是 IPC

IPC 是 Inter-Process Communication，也就是进程间通信。Linux 中每个进程默认有独立地址空间，一个进程不能直接访问另一个进程的内存。如果多个进程需要协作，就需要 IPC。

常见 IPC 方式：

  • 管道 pipe。
  • 命名管道 FIFO。
  • 信号 signal。
  • 共享内存 shared memory。
  • 消息队列 message queue。
  • Unix Domain Socket。
  • TCP/UDP socket。

一、什么是嵌入式开发

嵌入式开发是围绕特定硬件设备进行的软件开发。和普通 PC 或服务器开发相比，嵌入式更关注硬件资源、实时性、功耗、启动速度、稳定性和长期运行。

常见设备：

  • 路由器。
  • 工业控制器。
  • 智能家居设备。
  • 车载设备。
  • 摄像头。
  • 网关。
  • 可穿戴设备。

一、网络编程先理解什么

Linux 网络编程的核心是 socket。socket 是应用程序访问网络的接口，HTTP、WebSocket、RPC 等上层协议最终都建立在 socket 之上。

入门需要先理解：

  • IP：主机地址。
  • 端口：进程提供服务的入口。
  • TCP：可靠字节流。
  • UDP：无连接数据报。
  • 服务端：监听端口，等待连接。
  • 客户端：主动连接服务端。

一、系统编程学什么

Linux 系统编程关注应用程序和操作系统之间的接口。普通业务代码更多关心业务逻辑，系统编程则关心程序如何使用文件、进程、线程、信号、内存和网络。

入门阶段重点掌握：

  • 文件描述符。
  • open、read、write、close。
  • 进程创建和等待。
  • 线程创建和同步。
  • 信号处理。
  • 错误码和 errno。

一、Linux 应用开发是什么

Linux 应用开发通常指在 Linux 系统上编写、编译、调试、运行和发布用户态程序。它和 Android、嵌入式、服务端开发都有交集，但核心基础基本一致：

  • 熟悉 Linux 文件系统、权限、进程、线程、信号、网络等机制。
  • 能使用 GCC、Make、CMake 等工具构建程序。
  • 能使用 GDB、strace、lsof、top、journalctl 等工具定位问题。
  • 能理解动态库、静态库、环境变量、系统调用和进程通信。
  • 能把程序打包、部署，并在目标环境中稳定运行。
