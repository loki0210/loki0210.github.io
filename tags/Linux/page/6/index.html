一、为什么要服务化

  • 开机自动启动。
  • 异常退出自动重启。

  • 控制启动顺序。
  • 优雅停止。

Linux 现代发行版通常使用 systemd 管理服务。

一、为什么需要 epoll

阻塞 socket 服务端一次只能等待一个连接或一个读写事件。客户端数量少时问题不大，但连接数变多后，多进程和多线程模型会带来上下文切换、内存占用和调度开销。

Linux 下高并发网络服务常用 epoll。

epoll 适合：

  • 大量连接。
  • 多数连接空闲。
  • 事件驱动服务端。
  • 长连接服务。

一、什么是 IPC

IPC 是 Inter-Process Communication，也就是进程间通信。Linux 中每个进程默认有独立地址空间，一个进程不能直接访问另一个进程的内存。如果多个进程需要协作，就需要 IPC。

常见 IPC 方式：

  • 管道 pipe。
  • 命名管道 FIFO。
  • 信号 signal。
  • 共享内存 shared memory。
  • 消息队列 message queue。
  • Unix Domain Socket。
  • TCP/UDP socket。

  • 06/21
  • 11:32

一、为什么需要设备树

在嵌入式 Linux 中，内核需要知道硬件板子上有什么设备、这些设备接在哪些总线上、使用哪些 GPIO、中断、时钟和寄存器地址。

这些硬件描述不适合全部写死在驱动代码里，所以 Linux 使用设备树描述硬件。

设备树的作用：

  • 描述硬件资源。
  • 把板级差异从驱动中分离出来。
  • 让同一个驱动适配不同板子。
  • 让内核启动时创建设备。

一、网络编程先理解什么

Linux 网络编程的核心是 socket。socket 是应用程序访问网络的接口，HTTP、WebSocket、RPC 等上层协议最终都建立在 socket 之上。

入门需要先理解：

  • IP：主机地址。
  • 端口：进程提供服务的入口。
  • TCP：可靠字节流。
  • UDP：无连接数据报。
  • 服务端：监听端口，等待连接。
  • 客户端：主动连接服务端。

一、系统编程学什么

Linux 系统编程关注应用程序和操作系统之间的接口。普通业务代码更多关心业务逻辑，系统编程则关心程序如何使用文件、进程、线程、信号、内存和网络。

入门阶段重点掌握：

  • 文件描述符。
  • open、read、write、close。
  • 进程创建和等待。
  • 线程创建和同步。
  • 信号处理。
  • 错误码和 errno。
